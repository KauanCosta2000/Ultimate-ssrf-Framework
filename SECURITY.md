Security Policy
Responsible Disclosure

Please do not report security vulnerabilities through public GitHub issues.

If you discover a vulnerability related to this project, please report it responsibly so it can be reviewed and fixed before public disclosure.

How to Report a Vulnerability

Send the report directly to the maintainer through GitHub:

Maintainer: Kauan Costa
GitHub: https://github.com/KauanCosta2000

Please include as much detail as possible:

Vulnerability description
Impact
Affected file, function, or component
Steps to reproduce
Proof of Concept, if available
Logs, screenshots, or error output
Suggested fix, if known
Scope

Security reports may include:

Code execution issues
Unsafe file handling
Secrets or credential exposure
Dependency vulnerabilities
Unsafe AI provider handling
Unsafe report generation
SSRF scanner behavior that could unintentionally affect unauthorized systems
Out of Scope

The following are generally out of scope:

Reports without technical impact
Social engineering
Denial-of-service against public services
Issues caused by misuse of the tool
Vulnerabilities in third-party targets scanned without authorization
Disclosure Process
Report the issue privately.
The maintainer will review the report.
If valid, a fix will be prepared.
Public disclosure should only happen after a fix is available.
Safe Usage Notice

Ultimate SSRF Arsenal is intended only for:

Authorized penetration testing
Bug bounty programs
Security research
Educational environments

Do not use this tool against systems you do not own or do not have explicit permission to test.
